🔒 CSRF 保護測試工具

快速測試

請先確保您已經登入系統，然後點擊下方按鈕進行測試。

詳細測試項目

1. GET 請求不需要 CSRF token 待測試

測試 GET /api/users 請求，應該正常運作且不需要 CSRF token。

2. POST 請求沒有 CSRF token 應該被拒絕 待測試

測試 POST /api/users 請求（沒有 token），應該收到 403 錯誤。

3. POST 請求有 CSRF token 應該正常運作 待測試

測試 POST /api/users 請求（有 token），應該正常運作或收到業務邏輯錯誤（不是 CSRF 錯誤）。

4. PUT 請求沒有 CSRF token 應該被拒絕 待測試

測試 PUT 請求（沒有 token），應該收到 403 錯誤。

5. DELETE 請求沒有 CSRF token 應該被拒絕 待測試

測試 DELETE 請求（沒有 token），應該收到 403 錯誤。

6. 登入路由不需要 CSRF token 待測試

測試 POST /api/auth/login 請求（沒有 token），應該正常運作（登入成功或失敗，但不是 CSRF 錯誤）。

7. 前端自動取得 CSRF token 待測試

測試是否可以成功取得 CSRF token。
